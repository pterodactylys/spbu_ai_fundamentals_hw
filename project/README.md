## Прогноз цен смартфонов на Авито

### Abstract

В проекте рассматривается задача оценки рыночной стоимости б/у смартфонов по объявлениям Авито для города Санкт-Петербург. Цель проекта состоит в построении модели, которая по данным объявления предсказывает справедливую цену смартфона и позволяет выявлять выгодные предложения, где запрашиваемая цена заметно ниже предсказанной моделью. Использованы классические методы машинного обучения (Ridge, деревья решений, случайный лес, CatBoost, LightGBM); качество оценено по MAPE на 20% от всей выборки (7793 объявления после очистки).

### Goal and Metrics

* Цель - предсказание цены смартфона и поиск выгодных предложений на рынке б/у смартфонов.
* Дополнительная прикладная цель - ранжирование объявлений по степени выгодности покупки.
* Основная метрика оценки - MAPE.
* Дополнительная метрика для интерпретации ошибок - MAE в рублях.

### Data

Данные получены путем парсинга объявлений о продаже смартфонов по городу Санкт-Петербург на Авито (окно сбора ~14 дней, основные бренды: Apple, Samsung, Xiaomi, Huawei, Honor, Realme, OnePlus, Poco, Vivo и др.).

Основные признаки:

1. Производитель смартфона (brand)
2. Модель смартфона (model)
3. Объем встроенной памяти
4. Объем оперативной памяти
5. Состояние устройства
6. Наличие комплекта, коробки или чека, если это указано в объявлении
7. Текст заголовка и описания
8. Цена объявления
9. Дата публикации объявления
12. Количество фотографий
13. Данные о продавце.

Перед обучением данные очищены от дублей, пустых и некорректных объявлений; цена отфильтрована в диапазоне 500–200 000 ₽. Итоговый датасет — **7793** объявления (`data/smartphones_avito.csv`). По брендам выборка смещена в сторону Samsung (~78%) и Apple (~14%), что отражает структуру рынка и сбора на Авито.

### Validation Draft

Проверка предсказаний модели осуществлялась на отложенной выборке **20%**. Изначально планировалось разбиение по времени публикации; в реализации применено случайное разбиение train/test для сопоставимости моделей в ноутбуке и CLI-скриптах. Целевая переменная — `log1p(цена)`, метрики MAPE/MAE/RMSE считаются в рублях после обратного преобразования. Основная метрика оценки — MAPE.

### Approach

В качестве baseline-модели использована **Ridge-регрессия** с преобразованием признаков.

Полный список признаков для модели:

1. brand
2. storage_gb
3. ram_gb
4. condition
5. iphone_gen, is_pro, is_max, is_mini, is_plus
6. has_box
7. has_charger
8. has_warranty, has_trade_in
9. district
10. photos_count
11. title_len
12. description_len
13. days_since_posted, is_promoted
14. TF-IDF по заголовку (и опционально title+description для Ridge)

Для табличных признаков использованы медианное заполнение пропусков, кодирование категориальных переменных (OneHotEncoder / native categorical в бустинге) и фильтрация цены. Для текста сравнены TF-IDF только по названию и по названию+описанию; для бустинга добавлен TF-IDF по заголовку (300 признаков).

После baseline протестированы более сильные модели: **Decision Tree**, **Random Forest**, **CatBoost**, **LightGBM**.

### Training

Помимо baseline-модели, обучены альтернативные модели с разными наборами признаков и гиперпараметров. Сравнение включает:

1. Ridge
2. CatBoostRegressor
3. LightGBMRegressor

Сводка MAPE на hold-out (7793 строки, 20% test):

| Модель | MAPE | MAE |
|--------|------|-----|
| Dummy (mean) | 207% | 17 712 ₽ |
| LightGBM | **9.1%** | 1 213 ₽ |
| CatBoost | 20.3% | 2 813 ₽ |
| Ridge + TF-IDF (title) | 25.8% | 3 588 ₽ |
| Decision Tree | 24.0% | 2 929 ₽ |
| Random Forest | 25.1% | 3 187 ₽ |

Для бустинга использован early stopping (60 раундов). Отдельно проведён анализ ошибок по сегментам брендов (Apple / Samsung / остальные): у LightGBM MAPE на test — 14.7% / 4.6% / 39.3% соответственно.

### Validation

Модели сравнивались по MAPE и MAE на тестовой выборке. На сегменте Samsung и iPhone модель работает стабильнее; на редких брендах ошибка существенно выше.

### Model Selection

Финальная модель — **LightGBM** (MAPE 9.1% на, лучший баланс качества и скорости обучения). CatBoost показывает сопоставимое качество на Apple и устойчивее на отдельных объявлениях с нестандартной ценой. Основные риски подхода:

1. Неполные или шумные данные из объявлений.
2. Сильная зависимость цены от редких характеристик, которые могут быть плохо представлены в данных.
3. Изменение рынка со временем, из-за которого модель может устаревать и требовать регулярного переобучения.

### Deploy

Используемая в проекте модель может быть использована для поиска выгодных предложений в реальном времени. Точность предсказания зависит не только от качества модели, но и от актуальности данных, на которых производилось обучение. Для более точных предсказаний рекомендуется использовать данные не более чем **14-дневной** давности (как при сборе текущего датасета).
